実装者やAIが「なぜこの設計か」を理解し、将来の意思決定の文脈として参照します。

## ドキュメント構成

詳細は各ファイルに分割されています。実装タスクでは参照頻度を目安にファイルを選択してください。

| ファイル                                 | 内容                                                         | AI参照頻度                 |
| ---------------------------------------- | ------------------------------------------------------------ | -------------------------- |
| [詳細設計](design/detailed-design.md)    | データモデル・API仕様・エラーハンドリング                    | 高（実装時に必ず参照）     |
| [実装方針](design/implementation.md)     | 技術スタック・ディレクトリ構成・コーディング規約・テスト方針 | 高（実装時に必ず参照）     |
